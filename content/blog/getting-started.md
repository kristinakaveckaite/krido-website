This is a starter post so you can see how the blog looks and works. To publish a new article, add a `.md` file to `content/blog/` with the fields at the top (title, date, excerpt, optional coverImage), then write your post below in Markdown.

## Writing in Markdown

Use `##` for section headings, `**bold**` for emphasis, and `-` for bullet lists. Links look like `[text](https://example.com)`.

When you push the file to GitHub, Vercel rebuilds and your post goes live automatically — no need to touch any other file.

## Two ways to write posts

1. **In GitHub** — open the `content/blog` folder on github.com, click "Add file," and write directly in the browser.
2. **On your computer** — create the file in a text editor and push with git.

If you'd prefer a visual editor where you log in and type without files, let your developer know and a CMS can be connected instead.
